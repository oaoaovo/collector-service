
**重构目标**

在保持功能不变的前提下，降低重复代码、拆分过长函数、明确模块职责，让后续 HTTP API、采集任务、数据库操作和 JSON 处理更容易测试和维护。

**总体原则**

1. 每一步都保持可编译、可运行。
2. 优先做低风险公共能力抽取，再做业务模块拆分。
3. 旧接口尽量保留一段时间，避免一次性大改导致连锁问题。
4. 每个阶段完成后执行构建和核心手工验证。
5. 不改变现有 HTTP API、命令 JSON、数据库表结构和采集行为。

**阶段 1：公共 util 抽取**

目标：消除重复工具函数。

执行内容：

   - 提供 `escapeJson`
   - 提供基础 JSON 数组拼接辅助
   - 后续可扩展 `quote`、`boolText`、`arrayOf`

2. 新增或移动 `util/Logger`
   - 当前 `Logger` 已独立，但建议归入 `util`
   - 修改 include 路径为 `util/Logger.h`

3. 新增 `util/TimeUtil`
   - 抽出 `nowSeconds`
   - 抽出毫秒时间戳生成逻辑

涉及文件：

**阶段 3：HttpServer 拆分**

目标：让 `HttpServer` 只做 HTTP 连接处理，路由和 JSON 响应独立。

执行内容：

1. 新增 `http/HttpResponse`
   - 抽出 `jsonSuccess`
   - 抽出 `jsonError`
   - 抽出 Beast response 构造逻辑

2. 新增 `http/HttpJsonSerializer`
   - 抽出 `Device` 序列化
   - 抽出 `DeviceStatus` 序列化
   - 抽出 `ResourceRecord` 序列化

3. 新增 `http/HttpCommandAdapter`
   - 抽出 `commandEnvelope`

4. 新增 `http/HttpRouter`
   - 抽出原 `handleSession` 中的长 if-else
   - 每条路由一个小方法：
     - `handleHealth`
     - `handleDevices`
     - `handleStatus`
     - `handleLatestResources`
     - `handleCommands`
     - `handleTaskStart`
     - `handleTaskStop`

5. `HttpServer::handleSession`
   - 只负责读请求
   - 调用 `HttpRouter::route`
   - 写响应

**阶段 4：DataTask 拆分**

目标：拆分采集任务的大函数，降低线程、状态、解析、清理逻辑的耦合。

执行内容：

1. 新增 `CollectionQueryBuilder`
   - 抽出 `buildQuery`
   - 抽出 `makeCid`

2. 新增 `CollectionResponseParser`
   - 抽出 `extractObjectForPath`
   - 抽出 `extractStringField`
   - 抽出 `extractArrayFieldText`
   - 抽出 `extractNumberField`
   - 抽出 `extractBoolField`
   - 抽出 `processResponse` 的数据转换逻辑

3. 新增 `DeviceStatusStore`
   - 管理 `statuses_`
   - 管理 `activeDevices_`
   - 管理 stop flag 查询和状态设置

4. 可选新增 `DeviceCollectionWorker`

5. `DataTask` 保留 facade 职责：
   - `startAll`
   - `stopAll`
   - `startDeviceCollect`
   - `stopDeviceCollect`
   - `initDataPointsByDevice`

**阶段 5：SQLiteManager 拆分**

目标：把数据库管理和数据操作分开，降低 `SQLiteManager` 体积。

执行内容：

1. 新增 `SQLiteConnection`
   - 打开数据库
   - 关闭数据库
   - 暴露原始 sqlite handle

2. 新增 `SQLiteStatement`
   - 替代当前匿名 namespace 中的 `Statement`
   - 提供 bind、step、column 读取辅助

3. 新增 `SQLiteTransaction`
   - RAII 管理 `BEGIN/COMMIT/ROLLBACK`

4. 新增 repositories：
   - `DeviceRepository`
   - `DataPointRepository`
   - `ResourceRepository`

5. 新增 `ResourceRetentionService`
   - 负责 `cleanupResources`
   - 负责 `cleanupResourcesForDevice`

6. `SQLiteManager` 暂时保留为 facade：
   - 对外方法名不变
   - 内部委托给 repository/service
   - 这样可以避免一次性修改所有调用方

验证重点：


**风险点**

1. 当前 JSON 解析是手写字符串查找，抽取时容易改变边界行为，建议第一轮只搬迁，不重写逻辑。
2. `DataTask` 涉及线程生命周期，拆分时必须保持 stop flag、join、activeDevices 清理顺序不变。
3. `SQLiteManager` 使用 `recursive_mutex`，拆 repository 时要统一锁边界，避免死锁或锁粒度变化。
4. `HttpServer` 当前 route 行为是 API 兼容核心，拆路由时必须逐条保留返回体结构。
5. 文档存在乱码和过期边界，建议最后统一修，不要在代码拆分中夹杂大量文档变更。
